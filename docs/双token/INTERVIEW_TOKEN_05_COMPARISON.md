## 三、各方案详解

### 3.1 单Token JWT

```java
// 实现
String token = Jwts.builder()
    .setSubject(userId)
    .setExpiration(new Date(System.currentTimeMillis() + 7200000))
    .signWith(key)
    .compact();
```

**优点**：
- 实现简单
- 无状态，扩展性好
- 前后端分离友好

**缺点**：
- 无法主动吊销
- 有效期难平衡（太长不安全，太短体验差）

**适用**：简单应用、小程序、短期活动

---

### 3.2 双Token JWT（本项目方案）

**优点**：
- 安全性高（Access Token短期）
- 体验好（Refresh Token长期）
- 可控性强（可吊销Refresh Token）

**缺点**：
- 实现复杂
- 需要Redis存储
- 需要处理并发刷新

**适用**：企业后台、管理系统、金融应用

---

### 3.3 Session + Cookie

```java
// 传统方式
HttpSession session = request.getSession();
session.setAttribute("userId", userId);
session.setMaxInactiveInterval(7200); // 2小时
```

**优点**：
- 技术成熟
- 可实时吊销（删除Session即可）
- 浏览器原生支持

**缺点**：
- 有状态，扩展性差（需要共享Session）
- 跨域复杂
- 移动端支持差
- CSRF风险

**适用**：传统Web应用、内部系统

---

### 3.4 OAuth2.0

```
┌─────────┐                                    ┌─────────┐
│  用户   │─────1. 授权───────────────────────▶│  授权服 │
│         │◀────2. 授权码──────────────────────│  务器   │
│         │                                    │         │
│         │─────3. 授权码+密钥─────────────────▶│         │
│         │◀────4. Access/Refresh Token────────│         │
│         │                                    │         │
│         │─────5. Access Token───────────────▶│  资源服 │
│         │◀────6. 受保护资源──────────────────│  务器   │
└─────────┘                                    └─────────┘
```

**优点**：
- 标准化流程
- 支持第三方登录
- 权限粒度细

**缺点**：
- 流程复杂
- 实现成本高
- 需要维护授权服务器

**适用**：开放平台、第三方登录、多应用SSO

## 五、如何选择？

### 决策树

```
开始
 │
 ▼
是否需要第三方登录？
 │是
 └────────────────▶ OAuth2.0
 │否
 ▼
是否前后端分离？
 │否
 └────────────────▶ Session+Cookie
 │是
 ▼
是否需要长时间在线？
 │否（如外卖APP）
 └────────────────▶ 单Token
 │是（如后台管理）
 ▼
是否需要强制下线？
 │否
 └────────────────▶ 单Token（长有效期）
 │是
 ▼
是否有Redis？
 │否
 └────────────────▶ 单Token + 黑名单
 │是
 ▼
     双Token（推荐）
```

### 场景匹配

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 外卖/电商APP | 单Token 2小时 | 用完即走，重新登录简单 |
| 企业后台/ERP | 双Token | 长时间操作，需要可控性 |
| 银行/金融 | 双Token + 硬件Key | 最高安全级别 |
| 社交平台 | OAuth2.0 | 支持第三方登录 |
| 内部OA系统 | Session+Cookie | 简单可控，内网环境 |

## 七、未来趋势

### 7.1 WebAuthn（无密码认证）

```
使用指纹、面容识别代替密码
  │
  ▼
生物特征 ────▶ 本地验证 ────▶ 签名Token
```

### 7.2 JWT + Blockchain

```
去中心化身份验证
用户自己保管私钥
```

### 7.3 Zero Trust（零信任）

```
每次请求都验证
不区分内外网
持续验证用户身份和设备健康度
```
